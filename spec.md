プロジェクト名

Residential Traffic Monitoring PoC（坂道 × 通学路 × 住宅街向け）

横浜市神奈川区・三ツ沢第25号線周辺の、
“坂道で速度超過が多発し、通学路でもある住宅街の道路” を対象とした PoC。

目的は 車・バイク・自転車の速度と交通量を非接触で計測し、警察や行政に提出可能なレポートを自動生成する仕組み を構築すること。

目的（Purpose）

速度超過の実態把握
・坂道で加速する車両、タクシー、他県ナンバーなどの速度分布を可視化。

危険挙動の把握
・高速で下る自転車・バイクの速度
・車両通行量のピーク帯
・通学時間帯の交通状況

対外提出用レポートの自動生成
・警察署（神奈川警察署）や区役所へ提出できる PDF レポート
・Zone30, スピード抑制策の検討材料にする

全国展開できる OSS 化
・似た問題を抱える地域がそのまま使える
・GitHub に公開する想定

現地条件（Context）

ロケーション：横浜市神奈川区・三ツ沢エリアの坂道

道路分類：生活道路＋通学路＋横浜駅へ抜け道 → 実質「住宅街」

交通事情

車が制限速度超過で頻繁に通過

坂道のため自転車が高速で下ってくる

タクシー・他県ナンバーが多く、ナビに誘導される

バスも通行

設置条件

自宅敷地から道路を俯瞰（高さ 3–4m）

WiFi あり

電源あり

雨除けすればノート PC も屋外設置可

カメラは購入予定

機能要件（Requirements）
1. オブジェクト検知

YOLOv8 or YOLOv11
検知対象

車（普通車／タクシー）

バイク

自転車

歩行者

2. 車両トラッキング

SORT / ByteTrack を想定

同一車両をフレーム間で追跡し速度を算出

3. 速度推定

方式：2点間距離 × 移動フレーム数

カメラ画角の固定

道路に対して視点が一定

事前に Google Maps などで距離キャリブレーション

単眼カメラで十分可能

出力例：

時刻

種別（car / bike / bicycle / pedestrian）

推定速度（km/h）

移動方向（上り / 下り）

信頼度

4. 交通量計測

1日 / 時間帯別の交通量

平日・休日比較

通学時間帯のピーク特定

5. レポート自動生成（PDF）

警察向けテンプレート

行政向け参照資料

図表（速度ヒストグラム・時系列）

主要ポイント（速度超過率、危険挙動数）

6. ダッシュボード（任意）

ローカル Web UI

速度分布 / 通行量 Heatmap

日別比較

PC

余っている Windows ノートを Ubuntu 化

GPU なしでも OK（軽量版 YOLO）

カメラ（候補）
推奨：TP-Link Tapo C320WS
項目	内容
解像度	2K QHD
夜間性能	カラーナイトビジョン
電源	有線（AC）
価格	約7,000〜8,000円
PoE対応	×

→ 安定・安価・画質十分。PoC向け最適。

比較：C320WS vs C510W（ソーラー対応）
項目	C320WS	C510W（ソーラー可能）
解像度	2K	2K
夜間性能	カラーナイト	カラー
稼働方式	AC電源	AC または ソーラー＋バッテリ
屋外安定性	◎	◎（ただしソーラーは天候依存）
価格	安い	やや高い
長時間録画	◎	バッテリー依存で△
結論

安定性重視 → C320WS

電源引けない場所 → C510W

→ 今回は C320WS がベスト。
